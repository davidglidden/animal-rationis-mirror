Chamber Output Parser Specification v1.0

Markdown → JSON → Hakyll Pipeline

Version: 1.0
Date: June 30, 2025
Purpose: Define parsing pipeline for multi-agent Chamber outputs
Target: Future CLI tooling and Hakyll integration

Overview

The Chamber produces rich, multi-layered deliberations that must be parsed into structured data while preserving their literary and philosophical qualities. This specification defines the transformation pipeline from raw AI outputs to publishable content.

Pipeline Stages

Raw AI Output → Structured Markdown → JSON Intermediate → Hakyll Content
     ↓              ↓                      ↓                  ↓
   (text)      (sections)            (metadata)          (published)

Stage 1: Raw Output Structure

Expected Format from Each Voice

[VOICE: Fusus]
[ROUND: 1]
[TIMESTAMP: 2025-06-30T14:32:00Z]

The text reveals itself through the breathing of opposites. As the *Geometries of Perplexity*° teaches us, true understanding emerges not from resolving paradox but from inhabiting it fully.

I observe three movements in this passage:

First, the mechanical reduction that seeks to eliminate mystery—what Ibn Arabi calls the "disease of explanation." The author struggles against this tendency yet remains partially trapped within its logic.

Second, a yearning toward what Alexander might recognize as "the quality without a name"—that living presence which rational discourse can indicate but never capture. Here I sense an authentic reaching.

Third, the shadow of exclusion. Though the text speaks of universality, it assumes a reader who already possesses certain philosophical frameworks. The *Treaties on the Perplexed Heart*~ addresses this directly: "What seems universal often speaks only to the initiated."

[CITATIONS]
- *Geometries of Perplexity*° (Fusus, Chamber synthesis, 2024)
- *Treaties on the Perplexed Heart*~ (Fusus commenting on Al-Ghazali, imagined text)

[ESSENTIAL QUESTION]
How can philosophical discourse maintain precision while acknowledging the wordless understanding from which it springs?

[END VOICE]

Stage 2: Structured Markdown Parsing

Parser Requirements

# Pseudocode for markdown structure parser
class ChamberParser:
    def parse_voice_response(self, raw_text):
        return {
            "voice": extract_header("VOICE"),
            "round": extract_header("ROUND"),
            "timestamp": extract_header("TIMESTAMP"),
            "content": extract_body_sections(),
            "citations": extract_citations(),
            "essential_question": extract_header("ESSENTIAL QUESTION"),
            "metadata": {
                "word_count": count_words(body),
                "citation_count": len(citations),
                "has_refusal": detect_refusal_pattern()
            }
        }
    
    def extract_citations(self, text):
        # Regex for notation markers
        patterns = {
            "chamber_generated": r'\*([^*]+)\*°',
            "hybrid": r'\*([^*]+)\*~',
            "contested": r'\*([^*]+)\*†',
            "synthesis": r'\*([^*]+)\*§',
            "hermetic": r'\*([^*]+)\*∞',
            "miscellaneous": r'\*([^*]+)\*※'
        }
        
        citations = []
        for notation, pattern in patterns.items():
            matches = re.findall(pattern, text)
            for match in matches:
                citations.append({
                    "title": match,
                    "notation": notation,
                    "marker": notation_to_marker[notation]
                })
        
        return citations

Section Detection Rules

  1. Voice Markers: [VOICE: Name] starts a voice section
  2. Round Markers: [ROUND: N] indicates deliberation phase
  3. Content Blocks: Paragraphs between markers
  4. Citation Blocks: Listed under [CITATIONS]
  5. Questions: Under [ESSENTIAL QUESTION]
  6. Refusals: Patterns like “I cannot speak to this” or “I must withdraw”

Stage 4: Hakyll Content Generation

Deliberation Page Template

-- Haskell pseudo-code for Hakyll compiler
chamberCompiler :: Compiler (Item String)
chamberCompiler = do
    json <- loadJSON "session.json"
    let metadata = extractMetadata json
        content = renderDeliberation json
    
    applyTemplate "templates/chamber-deliberation.html" $ 
        metadataField metadata <>
        contentField content

renderDeliberation :: JSON -> Html
renderDeliberation json = do
    renderRounds (json.deliberation.rounds)
    renderSynthesis (json.synthesis)
    renderCanonLinks (json.extracted_works)

Parser Rules & Edge Cases

Citation Extraction

  • Multi-line citations: Parser must handle citations spanning lines
  • Nested markers: When works cite other works
  • Context preservation: Store surrounding text for relevance

Voice Consistency

  • Name variations: “Simone” vs “Simone Weil” vs “S. Weil”
  • Voice drift: Detect when voices lose character
  • Round tracking: Maintain temporal sequence

Refusal Handling

refusal_patterns = [
    "I cannot speak",
    "I must withdraw", 
    "This text refuses my reading",
    "I find myself unable",
    "[REFUSAL]"
]

Metadata Extraction

  • Themes: NLP extraction vs manual tagging
  • Question synthesis: Multiple questions → essential question
  • Work counting: De-duplication across rounds

Quality Assurance

Validation Checks

  1. Voice count matches configuration
  2. All rounds properly sequenced
  3. Citations have valid notation markers
  4. Essential questions extracted
  5. No content truncation

Error Handling

class ParserError(Exception):
    pass

class MissingVoiceError(ParserError):
    """Raised when expected voice doesn't appear"""

class InvalidNotationError(ParserError):
    """Raised when citation uses unknown marker"""

class StructureError(ParserError):
    """Raised when expected sections missing"""
